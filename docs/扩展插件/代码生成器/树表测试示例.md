# 树表结构设计示例

## 场景说明

以**部门管理**为例，展示树表的完整结构设计。

树表特点：
- 每个节点可以有多个子节点
- 通过 `parent_id` 字段建立父子关系
- 支持无限层级
- 提供树形结构查询、移动节点等功能

---

## 1. 数据库表设计

### 部门树表 (sys_department)

```sql
CREATE TABLE sys_department (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '部门ID',
    parent_id BIGINT DEFAULT 0 COMMENT '父部门ID（0表示根节点）',
    dept_name VARCHAR(100) NOT NULL COMMENT '部门名称',
    dept_code VARCHAR(50) COMMENT '部门编码',
    leader VARCHAR(50) COMMENT '负责人',
    phone VARCHAR(20) COMMENT '联系电话',
    email VARCHAR(100) COMMENT '邮箱',
    sort_no INT DEFAULT 0 COMMENT '显示顺序',
    status INT DEFAULT 0 COMMENT '状态（0正常 1停用）',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_by BIGINT COMMENT '创建人',
    update_by BIGINT COMMENT '更新人',
    remark VARCHAR(500) COMMENT '备注',

    INDEX idx_parent_id (parent_id),
    INDEX idx_sort_no (sort_no)
) COMMENT='部门树表';
```

### Service 接口

```java
public interface DepartmentService {

    /**
     * 查询所有部门列表（平铺）
     */
    List<Department> findAll();

    /**
     * 根据ID查询部门
     */
    Department findById(Long id);

    /**
     * 🔥 核心方法：查询树形结构
     */
    List<Department> findTree();

    /**
     * 🔥 核心方法：根据父节点ID查询子树
     */
    List<Department> findTreeByParentId(Long parentId);

    /**
     * 创建部门
     */
    Boolean create(DepartmentDto departmentDto);

    /**
     * 更新部门
     */
    Boolean update(DepartmentDto departmentDto);

    /**
     * 根据ID删除部门
     * 🔥 有子节点时不允许删除
     */
    Boolean deleteById(Long id);

    /**
     * 🔥 核心方法：移动节点
     */
    Boolean moveNode(Long id, Long newParentId);
}
```

## 5. Controller 接口

```java
@RestController
@RequestMapping("/system/department")
public class DepartmentController {

    @Autowired
    private DepartmentService departmentService;

    /**
     * 查询部门列表（平铺）
     */
    @GetMapping("/list")
    public Result<List<Department>> list() {
        List<Department> list = departmentService.findAll();
        return Result.success(list);
    }

    /**
     * 🔥 查询部门树
     */
    @GetMapping("/tree")
    public Result<List<Department>> tree() {
        List<Department> tree = departmentService.findTree();
        return Result.success(tree);
    }

    /**
     * 🔥 根据父节点ID查询子树
     */
    @GetMapping("/tree/{parentId}")
    public Result<List<Department>> treeByParentId(@PathVariable Long parentId) {
        List<Department> tree = departmentService.findTreeByParentId(parentId);
        return Result.success(tree);
    }

    /**
     * 根据ID查询部门
     */
    @GetMapping("/{id}")
    public Result<Department> getById(@PathVariable Long id) {
        Department department = departmentService.findById(id);
        return Result.success(department);
    }

    /**
     * 创建部门
     */
    @PostMapping
    public Result<Boolean> create(@RequestBody DepartmentDto departmentDto) {
        Boolean flag = departmentService.create(departmentDto);
        return Result.success(flag);
    }

    /**
     * 更新部门
     */
    @PutMapping
    public Result<Boolean> update(@RequestBody DepartmentDto departmentDto) {
        Boolean flag = departmentService.update(departmentDto);
        return Result.success(flag);
    }

    /**
     * 删除部门
     */
    @DeleteMapping("/{id}")
    public Result<Boolean> delete(@PathVariable Long id) {
        Boolean flag = departmentService.deleteById(id);
        return Result.success(flag);
    }

    /**
     * 🔥 移动节点
     */
    @PutMapping("/move/{id}/{newParentId}")
    public Result<Boolean> moveNode(@PathVariable Long id, @PathVariable Long newParentId) {
        Boolean flag = departmentService.moveNode(id, newParentId);
        return Result.success(flag);
    }
}
```

## 7. 前端使用示例

### Vue 3 + Element Plus 树形表格

```vue
<template>
  <div>
    <el-button @click="loadTree">刷新部门树</el-button>

    <el-table
      :data="tableData"
      style="width: 100%; margin-top: 20px"
      row-key="id"
      :tree-props="{ children: 'children' }"
    >
      <el-table-column prop="deptName" label="部门名称" />
      <el-table-column prop="deptCode" label="部门编码" />
      <el-table-column prop="leader" label="负责人" />
      <el-table-column prop="sortNo" label="排序" />
      <el-table-column label="操作" width="300">
        <template #default="{ row }">
          <el-button @click="handleAdd(row)">添加子部门</el-button>
          <el-button @click="handleEdit(row)">编辑</el-button>
          <el-button @click="handleDelete(row)">删除</el-button>
          <el-button @click="handleMove(row)">移动</el-button>
        </template>
      </el-table-column>
    </el-table>
  </div>
</template>

<script setup>
import { ref } from 'vue';
import axios from 'axios';

const tableData = ref([]);

// 加载部门树
const loadTree = async () => {
  const response = await axios.get('/system/department/tree');
  tableData.value = response.data.data;
};

// 添加子部门
const handleAdd = (row) => {
  console.log('添加子部门到：', row.deptName);
  // 打开对话框，parentId 默认为 row.id
};

// 编辑部门
const handleEdit = (row) => {
  console.log('编辑部门：', row.deptName);
};

// 删除部门
const handleDelete = async (row) => {
  try {
    await axios.delete(`/system/department/${row.id}`);
    alert('删除成功');
    loadTree();
  } catch (error) {
    alert('删除失败：' + error.response.data.message);
  }
};

// 移动节点
const handleMove = async (row) => {
  const newParentId = prompt('请输入新的父节点ID');
  if (newParentId) {
    try {
      await axios.put(`/system/department/move/${row.id}/${newParentId}`);
      alert('移动成功');
      loadTree();
    } catch (error) {
      alert('移动失败：' + error.response.data.message);
    }
  }
};

// 初始化
loadTree();
</script>
```

## 10. 树表的注意事项

⚠️ **数据库设计**
- 必须有 `parent_id` 字段（或其他指定的父字段）
- 建议添加 `sort_no` 字段用于排序
- 建议为 `parent_id` 添加索引以提高查询性能

⚠️ **业务逻辑**
- 根节点的 `parent_id` 通常为 0 或 null
- 删除节点前需要检查是否有子节点
- 移动节点时需要防止循环引用

⚠️ **性能优化**
- 对于大量数据，建议使用缓存
- 可以添加左右值编码（闭包表）来优化查询
- 深度限制：建议不要超过 5 层

⚠️ **前端展示**
- 使用树形组件（如 Element Plus 的 el-table）
- 支持展开/折叠
- 支持拖拽移动节点
