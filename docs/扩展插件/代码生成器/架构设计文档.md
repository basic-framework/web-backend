## 1. 项目概述

### 1.1 简介

智领代码生成器是一个基于数据库表结构自动生成Java后端代码的插件工具。它支持单表、主子表、树表等多种表类型的代码生成，可以大幅提高开发效率，减少重复劳动。

### 1.2 核心特性

- ✅ **多表类型支持**：单表、主子表、树表
- ✅ **可视化配置**：Web界面配置生成规则
- ✅ **智能代码生成**：自动识别主键、字段类型、关联关系
- ✅ **模板化生成**：基于Velocity模板引擎
- ✅ **代码预览**：生成前可预览代码效果
- ✅ **批量生成**：支持一次性生成多张表代码
- ✅ **字段过滤**：自动过滤BaseEntity基础字段
- ✅ **分层架构**：生成标准的Controller-Service-Mapper三层架构代码

### 1.3 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Spring Boot | 2.x | 基础框架 |
| MyBatis | 3.x | ORM框架 |
| MyBatis-Plus | 3.x | MyBatis增强工具 |
| Velocity | 2.3 | 模板引擎 |
| Hutool | 5.x | 工具类库 |
| Lombok | 1.18.x | 简化Java代码 |
| MySQL | 8.0+ | 数据库 |
| Swagger | 3.0 | API文档 |

## 3. 功能特性

### 3.1 表类型支持

#### 3.1.1 单表（SINGLE）

**适用场景**：普通的CRUD表

**生成内容**：
- Domain实体类
- DTO/VO/BO
- Mapper接口和XML
- Service接口和实现
- Controller

**特点**：标准的增删改查功能

#### 3.1.2 主表（MASTER）

**适用场景**：包含多个子表的主表（如订单-订单项）

**生成内容**：
- Domain实体类（包含子表列表）
- Service实现（级联查询、保存、更新、删除）
- 其他同单表

**特点**：
- 自动查询关联的子表数据
- 保存时自动保存子表
- 更新时自动更新子表
- 删除时自动删除子表

#### 3.1.3 子表（DETAIL）

**适用场景**：关联到主表的子表

**生成内容**：
- Domain实体类（包含主表对象引用）
- Mapper（包含根据主表ID查询的方法）
- 其他同单表

**特点**：
- 不生成独立的Controller
- 通过主表的Service进行操作
- 支持根据主表ID进行关联查询和删除

#### 3.1.4 树表（TREE）

**适用场景**：具有父子关系的树形结构表（如部门、菜单）

**生成内容**：
- Domain实体类（包含children字段）
- Service实现（树形结构操作）
- Controller（树形查询接口）
- 其他同单表

**特点**：
- `findTree()` - 查询整棵树
- `findTreeByParentId()` - 查询子树
- `moveNode()` - 移动节点
- 删除时检查是否有子节点
- 移动时防止循环引用

### 3.2 核心功能

#### 3.2.1 从数据库导入表

- 自动读取数据库表结构
- 自动识别字段类型
- 自动识别主键
- 可选择导入哪些表

#### 3.2.2 字段配置

- 可配置字段的显示/隐藏
- 可配置字段的操作权限（增删改查）
- 可配置查询方式（等于、不等于、模糊等）
- 可配置显示类型（输入框、下拉框、日期等）
- 支持批量配置

#### 3.2.3 代码生成

- 支持单表生成
- 支持批量生成
- 支持代码预览
- 支持下载ZIP包
- 自动按表类型选择模板

#### 3.2.4 主子表配置

- 可视化配置主子表关系
- 自动识别并加载关联的子表
- 自动生成级联操作代码

## 4. 技术实现

### 4.1 代码生成流程

```java
// 1. 从数据库读取表结构
DatabaseMetaData → 表信息、字段信息、主键信息

// 2. 转换为Schema和SchemaField
表名 → Schema
字段列表 → List<SchemaField>

// 3. 准备Velocity上下文
Schema + SchemaField + GenConfig → Map<String, Object> context

// 4. 选择模板
tableType → 模板列表（Map<String, String> templates）

// 5. 渲染模板
Velocity + context → 生成的代码

// 6. 输出文件
生成的代码 → 写入到指定目录或打包为ZIP
```

### 4.2 核心代码实现

#### 4.2.1 主子表处理

```java
// 加载主表的子表信息
if ("MASTER".equals(schema.getTableType())) {
    List<Schema> detailTables = schemaService.findByMasterTableId(masterTableId);

    // 为每个子表设置必要信息
    for (Schema detailTable : detailTables) {
        detailTable.setClassName(convertToClassName(detailTable.getTableName()));
        detailTable.setBusinessName(StrUtil.lowerFirst(detailTable.getClassName()));
        detailTable.setMasterTableName(schema.getTableName());
        detailTable.setMasterClassName(schema.getClassName());
    }

    schema.setDetailTables(detailTables);
}
```

#### 4.2.2 树表处理

```java
// 树表特殊处理
if ("TREE".equals(schema.getTableType())) {
    // 设置树表特殊字段
    context.put("treeParentField", schema.getTreeParentField());
    context.put("treeChildrenField", schema.getTreeChildrenField());
}
```

#### 4.2.3 字段过滤

```java
// 过滤BaseEntity字段
Set<String> baseEntityFields = new HashSet<>(Arrays.asList(
    "id", "create_time", "update_time",
    "create_by", "update_by", "remark", "status"
));

while (colRs.next()) {
    String columnName = colRs.getString("COLUMN_NAME");

    // 跳过BaseEntity字段
    if (baseEntityFields.contains(columnName.toLowerCase())) {
        continue;
    }

    // 处理业务字段
    fields.add(processField(columnName));
}
```

#### 4.3.2 模板选择逻辑

```java
public static Map<String, String> getTemplateList(String tableType) {
    Map<String, String> templates = new LinkedHashMap<>();

    switch (tableType) {
        case "MASTER":
            // 主表模板（包含子表操作）
            templates.put("domain.java.vm", "vm/java/domainMaster.java.vm");
            templates.put("serviceImpl.java.vm", "vm/java/serviceImplMaster.java.vm");
            break;

        case "DETAIL":
            // 子表模板（包含关联查询）
            templates.put("domain.java.vm", "vm/java/domainDetail.java.vm");
            templates.put("mapper.java.vm", "vm/java/mapperDetail.java.vm");
            templates.put("mapper.xml.vm", "vm/xml/mapperDetail.xml.vm");
            break;

        case "TREE":
            // 树表模板（包含树形操作）
            templates.put("domain.java.vm", "vm/java/domainTree.java.vm");
            templates.put("service.java.vm", "vm/java/serviceTree.java.vm");
            templates.put("serviceImpl.java.vm", "vm/java/serviceImplTree.java.vm");
            templates.put("controller.java.vm", "vm/java/controllerTree.java.vm");
            break;

        default:
            // 单表模板（默认）
            templates.put("domain.java.vm", "vm/java/domain.java.vm");
            templates.put("serviceImpl.java.vm", "vm/java/serviceImpl.java.vm");
            templates.put("controller.java.vm", "vm/java/controller.java.vm");
            break;
    }

    return templates;
}
```

### 5.3 配置主子表

#### 场景：订单（主表）+ 订单项（子表）

**步骤1：导入两张表**

导入 `sys_order` 和 `sys_order_item` 两张表

**步骤2：配置主表**

1. 选择 `sys_order` 表
2. 表类型选择：`MASTER（主表）`
3. 保存配置

**步骤3：配置子表**

1. 选择 `sys_order_item` 表
2. 表类型选择：`DETAIL（子表）`
3. 主表表名：`sys_order`
4. 关联字段名：`order_id`（子表中的外键字段）
5. 保存配置

**步骤4：生成代码**

点击"生成全部代码"，系统会：
- 为主表生成包含子表列表的完整代码
- 为子表生成包含关联查询的代码
- 自动处理级联操作

### 5.4 配置树表

**步骤1：导入表**

导入 `sys_department` 表（包含 `parent_id` 字段）

**步骤2：配置为树表**

1. 选择 `sys_department` 表
2. 表类型选择：`TREE（树表）`
3. 父字段名：`parent_id`
4. 子节点集合字段名：`children`
5. 保存配置

**步骤3：生成代码**

生成的代码包含：
- 树形结构查询方法
- 移动节点方法
- 删除保护（有子节点时不允许删除）

gen:

## 7. 模板系统

### 7.1 模板语法

Velocity模板使用 `#` 开头的指令：

```velocity
# 注释
## 这是单行注释

# 变量输出
${packageName}

# 条件判断
#if($condition)
    ...
#else
    ...
#end

# 循环
#foreach($item in $list)
    ${item.name}
#end

# 宏定义
#macro(methodName $param1 $param2)
    ...
#end
```

### 7.3 自定义模板

如需自定义模板，按照以下步骤：

**步骤1：创建模板文件**

 * @author ${author}

Map<String, String> templates = new LinkedHashMap<>();

if (templateName.equals("yourTemplate.java.vm")) {

## 11. 常见问题

### 11.1 生成的代码编译失败

**问题**：生成的代码有语法错误或编译错误

**原因**：
1. 模板语法错误
2. 字段类型映射错误
3. 包名配置错误

**解决**：
1. 检查模板文件语法
2. 检查数据库字段类型映射
3. 检查 `generator.yml` 配置

### 11.2 主子表代码不生效

**问题**：配置主子表后，生成的代码没有关联关系

**原因**：
1. 子表的 `master_table_id` 没有正确配置
2. 子表的 `relation_field` 配置错误
3. 代码生成器没有正确加载子表信息

**解决**：
1. 检查子表的配置是否正确
2. 确认 `relation_field` 是子表中的外键字段名
3. 查看日志确认是否正确加载子表
4. 重新生成代码

### 11.3 树表查询性能慢

**问题**：树表查询很慢

**原因**：
1. 递归查询效率低
2. 缺少索引
3. 数据量太大

**解决**：
1. 为 `parent_id` 字段添加索引
2. 使用缓存
3. 考虑使用闭包表或路径枚举优化
4. 限制树的深度

### 11.4 字段重复生成

**问题**：生成的实体类中字段重复

**原因**：
1. 数据库表字段与BaseEntity字段重复
2. 字段过滤配置不正确

**解决**：
1. 确认 `buildFieldsFromTable` 方法中的字段过滤列表
2. 检查数据库表是否包含BaseEntity的字段
3. 重新生成代码

### 11.5 模板找不到

**问题**：生成时报错找不到模板文件

**原因**：
1. 模板文件路径错误
2. 模板文件名错误

**解决**：
1. 检查 `vm/` 目录下是否存在对应的模板文件
2. 检查 `VelocityUtils.getTemplateList()` 中的路径配置
3. 确认模板文件名正确
