# zl-common-sensitive 数据脱敏模块详解

## 1. 模块概述

`zl-common-sensitive` 是智领软件公司基础脚手架中的一个数据脱敏模块，主要用于在数据序列化为JSON时对敏感信息进行自动脱敏处理。该模块基于Jackson的序列化机制，通过注解方式标记需要脱敏的字段，并支持多种脱敏策略。

### 1.1 核心功能

- **自动数据脱敏**：在JSON序列化过程中自动对标记的字段进行脱敏处理
- **多种脱敏策略**：支持身份证、手机号、邮箱、银行卡等多种常见敏感数据的脱敏
- **权限控制**：可根据用户权限决定是否进行脱敏处理
- **易于扩展**：支持自定义脱敏策略和脱敏规则

### 1.2 模块结构

```

├── src/main/java/com/zl/sensitive/
```

## 2. 核心组件详解

### 2.1 Sensitive 注解

[`Sensitive.java`](zl-common-core/zl-common-sensitive/src/main/java/com/zl/sensitive/annotation/Sensitive.java:1) 是数据脱敏的核心注解，用于标记需要进行脱敏处理的字段。

```java
@Retention(RetentionPolicy.RUNTIME)  // 运行时生效，允许反射解析
@Target(ElementType.FIELD)           // 仅作用于实体类字段
@JacksonAnnotationsInside            // 告诉Jackson：这是一个内部注解，需解析其绑定的序列化器
@JsonSerialize(using = SensitiveHandler.class)  // 指定该字段的序列化器为SensitiveHandler
public @interface Sensitive {
    SensitiveStrategy strategy();    // 必选：指定脱敏策略（手机号/身份证等）
}
```

**关键特性**：
- 使用 `@Retention(RetentionPolicy.RUNTIME)` 确保注解在运行时可用
- 使用 `@Target(ElementType.FIELD)` 限制注解只能用于字段
- 通过 `@JacksonAnnotationsInside` 和 `@JsonSerialize` 指定自定义的序列化器

### 2.2 SensitiveStrategy 脱敏策略

[`SensitiveStrategy.java`](zl-common-core/zl-common-sensitive/src/main/java/com/zl/sensitive/core/SensitiveStrategy.java:1) 是一个枚举类，定义了各种脱敏策略。

```java
@AllArgsConstructor
public enum SensitiveStrategy {
    /**
     * 身份证脱敏
     */
    ID_CARD(s -> DesensitizedUtil.idCardNum(s, 3, 4)),
    
    /**
     * 密钥脱敏
     */
    SKY(s -> DesensitizedUtil.idCardNum(s, 0, 1)),
    
    /**
     * 手机号脱敏
     */
    PHONE(DesensitizedUtil::mobilePhone),
    
    /**
     * 地址脱敏
     */
    ADDRESS(s -> DesensitizedUtil.address(s, 8)),
    
    /**
     * 邮箱脱敏
     */
    EMAIL(DesensitizedUtil::email),
    
    /**
     * 银行卡
     */
    BANK_CARD(DesensitizedUtil::bankCard);
    
    private final Function<String, String> desensitizer;
    
    public Function<String, String> desensitizer() {
        return desensitizer;
    }
}
```

**设计亮点**：
- 使用函数式接口 `Function<String, String>` 定义脱敏逻辑，提高灵活性
- 基于Hutool的 `DesensitizedUtil` 工具类实现具体脱敏算法
- 支持自定义脱敏参数，如身份证脱敏保留前3位和后4位

### 2.3 SensitiveService 脱敏服务接口

[`SensitiveService.java`](zl-common-core/zl-common-sensitive/src/main/java/com/zl/sensitive/core/SensitiveService.java:1) 是一个简单的服务接口，用于控制是否进行脱敏。

```java
public interface SensitiveService {
    /**
     * 是否脱敏
     */
    boolean isSensitive();
}
```

**设计思路**：
- 提供一个简单的布尔方法决定是否进行脱敏
- 默认管理员不过滤，需要根据业务重写实现
- 允许基于用户角色、权限或其他业务逻辑动态控制脱敏行为

### 2.4 SensitiveHandler 序列化处理器

[`SensitiveHandler.java`](zl-common-core/zl-common-sensitive/src/main/java/com/zl/sensitive/handler/SensitiveHandler.java:1) 是核心的JSON序列化处理器，负责实际的脱敏处理。

```java
@Slf4j
public class SensitiveHandler extends JsonSerializer<String> implements ContextualSerializer {
    
    private SensitiveStrategy strategy;
    
    @Override
    public void serialize(String value, JsonGenerator gen, SerializerProvider serializers) throws IOException {
        try {
            SensitiveService sensitiveService = SpringUtil.getBean(SensitiveService.class);
            if (ObjectUtil.isNotNull(sensitiveService) && sensitiveService.isSensitive()) {
                gen.writeString(strategy.desensitizer().apply(value));
            } else {
                gen.writeString(value);
            }
        } catch (BeansException e) {
            log.error("脱敏实现不存在, 采用默认处理 => {}", e.getMessage());
            gen.writeString(value);
        }
    }
    
    @Override
    public JsonSerializer<?> createContextual(SerializerProvider prov, BeanProperty property) throws JsonMappingException {
        Sensitive annotation = property.getAnnotation(Sensitive.class);
        if (Objects.nonNull(annotation) && Objects.equals(String.class, property.getType().getRawClass())) {
            this.strategy = annotation.strategy();
            return this;
        }
        return prov.findValueSerializer(property.getType(), property);
    }
}
```

**核心功能**：
- 实现 `JsonSerializer<String>` 接口，处理字符串类型的序列化
- 实现 `ContextualSerializer` 接口，支持根据注解动态配置序列化器
- 通过 `SpringUtil.getBean()` 获取脱敏服务，实现动态脱敏控制
- 提供异常处理机制，当脱敏服务不存在时返回原始值

## 3. 使用思路

### 3.1 基本使用

1. **在实体类字段上添加注解**：

```java
public class UserInfo {
    @Sensitive(strategy = SensitiveStrategy.PHONE)
    private String phoneNumber;
    
    @Sensitive(strategy = SensitiveStrategy.ID_CARD)
    private String idCard;
    
    @Sensitive(strategy = SensitiveStrategy.EMAIL)
    private String email;
    
    // 其他字段...
}
```

2. **实现SensitiveService接口**（可选）：

```java
@Service
public class CustomSensitiveService implements SensitiveService {
    
    @Override
    public boolean isSensitive() {
        // 获取当前用户信息
        UserContext user = UserContext.getCurrent();
        
        // 如果是管理员，不进行脱敏
        if (user.isAdmin()) {
            return false;
        }
        
        // 其他情况进行脱敏
        return true;
    }
}
```

3. **返回JSON数据**：

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @GetMapping("/{id}")
    public UserInfo getUserInfo(@PathVariable Long id) {
        UserInfo user = userService.getById(id);
        return user; // 自动进行脱敏处理
    }
}
```

### 3.2 高级使用

1. **自定义脱敏策略**：

```java
// 在SensitiveStrategy枚举中添加新策略
public enum SensitiveStrategy {
    // 现有策略...
    
    /**
     * 自定义姓名脱敏
     */
    NAME(s -> {
        if (StringUtils.isBlank(s)) {
            return s;
        }
        if (s.length() <= 2) {
            return s.charAt(0) + "*";
        }
        return s.charAt(0) + "*".repeat(s.length() - 2) + s.charAt(s.length() - 1);
    });
}
```

2. **基于角色的脱敏控制**：

```java
@Service
public class RoleBasedSensitiveService implements SensitiveService {
    
    @Override
    public boolean isSensitive() {
        UserContext user = UserContext.getCurrent();
        
        // 根据用户角色决定是否脱敏
        return !user.hasRole("ADMIN") && !user.hasRole("DATA_MANAGER");
    }
}
```

## 4. 详细设计思路

### 4.1 整体架构设计

该模块采用了**策略模式**和**模板方法模式**的组合设计：

1. **策略模式**：`SensitiveStrategy` 枚举定义了各种脱敏策略，每种策略对应一种脱敏算法
2. **模板方法模式**：`SensitiveHandler` 定义了脱敏处理的流程，具体策略由注解指定
3. **依赖注入**：通过 `SpringUtil` 获取 `SensitiveService` 实例，实现脱敏控制的动态化

### 4.2 核心流程分析

1. **注解解析阶段**：
   - Jackson在序列化过程中检测到字段上的 `@Sensitive` 注解
   - 调用 `SensitiveHandler.createContextual()` 方法创建上下文序列化器
   - 从注解中提取指定的脱敏策略

2. **序列化处理阶段**：
   - Jackson调用 `SensitiveHandler.serialize()` 方法
   - 通过 `SpringUtil.getBean()` 获取 `SensitiveService` 实例
   - 根据 `SensitiveService.isSensitive()` 决定是否进行脱敏
   - 如果需要脱敏，应用指定的脱敏策略；否则返回原始值

### 4.3 关键设计决策

1. **基于Jackson序列化机制**：
   - 优点：与Spring Boot无缝集成，支持REST API自动脱敏
   - 缺点：仅适用于JSON序列化场景

2. **使用函数式接口定义脱敏策略**：
   - 优点：灵活、简洁，支持Lambda表达式
   - 缺点：需要Java 8+支持

3. **通过SpringUtil获取服务实例**：
   - 优点：避免直接依赖注入，提高模块的独立性
   - 缺点：增加了对SpringUtil工具类的依赖

4. **提供SensitiveService接口**：
   - 优点：支持基于业务逻辑的动态脱敏控制
   - 缺点：需要用户自行实现，增加了使用复杂度

### 4.4 扩展性设计

1. **策略扩展**：
   - 可通过在 `SensitiveStrategy` 枚举中添加新值来扩展脱敏策略
   - 每种策略都是一个 `Function<String, String>`，支持自定义脱敏逻辑

2. **控制扩展**：
   - 可通过实现 `SensitiveService` 接口来自定义脱敏控制逻辑
   - 支持基于用户角色、权限、时间等多种因素的动态控制

3. **类型扩展**：
   - 当前仅支持 `String` 类型，可通过扩展 `SensitiveHandler` 支持其他类型

## 5. 最佳实践

### 5.1 使用建议

1. **合理选择脱敏策略**：
   - 根据数据类型和业务需求选择合适的脱敏策略
   - 对于自定义数据类型，考虑添加专门的脱敏策略

2. **实现合适的脱敏控制**：
   - 根据业务需求实现 `SensitiveService` 接口
   - 考虑基于用户角色、数据敏感级别等因素进行控制

3. **测试脱敏效果**：
   - 编写单元测试验证各种脱敏策略的效果
   - 测试不同用户角色下的脱敏行为

### 5.2 注意事项

1. **性能考虑**：
   - 脱敏处理会增加序列化时间，对性能敏感的场景需谨慎使用
   - 考虑缓存脱敏结果，避免重复计算

2. **安全性考虑**：
   - 脱敏仅适用于展示层，数据库中仍存储原始数据
   - 确保日志、缓存等环节不会泄露敏感信息

3. **兼容性考虑**：
   - 确保脱敏后的数据格式符合前端预期
   - 考虑API版本兼容性，避免脱敏规则变更影响客户端

## 6. 总结

`zl-common-sensitive` 模块是一个设计精良的数据脱敏解决方案，具有以下优点：

1. **易用性**：通过注解方式标记脱敏字段，使用简单直观
2. **灵活性**：支持多种脱敏策略，可自定义脱敏规则和控制逻辑
3. **集成性**：与Spring Boot和Jackson无缝集成，适用于REST API场景
4. **扩展性**：采用策略模式和函数式接口，易于扩展新的脱敏策略

该模块适用于需要对敏感数据进行脱敏处理的场景，特别是用户信息展示、日志记录、数据导出等场景。通过合理使用该模块，可以有效保护用户隐私，提高系统的安全性。
