## 1. 日志技术介绍

### 1.1 技术栈
本项目采用业界标准的日志技术栈：

- **SLF4J (Simple Logging Facade for Java)**: 日志门面，提供统一的日志接口
- **Logback**: 具体的日志实现，由SLF4J的作者开发，性能优异
- **Lombok**: 通过注解简化日志代码编写

### 1.2 技术优势

#### SLF4J优势
- **统一接口**: 提供统一的日志API，与具体实现解耦
- **性能优化**: 支持参数化日志，避免不必要的字符串拼接
- **灵活切换**: 可以轻松切换底层日志实现

#### Logback优势
- **性能卓越**: 比Log4j性能更好，内存占用更少
- **配置灵活**: 支持复杂的日志配置策略
- **功能丰富**: 支持异步日志、日志滚动、压缩等功能

#### Lombok优势
- **代码简化**: 通过`@Slf4j`注解自动生成日志实例
- **类型安全**: 编译时检查，避免运行时错误
- **开发效率**: 减少样板代码，提高开发效率

### 1.3 为什么不使用Log4j
- **性能考虑**: Logback性能优于Log4j
- **Spring Boot默认**: Spring Boot默认使用Logback
- **发展趋势**: Log4j已停止维护，Log4j2虽然性能好但配置复杂

## 2. 日志架构

```
应用程序代码
    ↓
Lombok @Slf4j 注解
    ↓
SLF4J 日志门面
    ↓
Logback 日志实现
    ↓
控制台/文件输出
```

### 2.1 组件关系
- **应用程序**: 通过Lombok注解使用日志
- **SLF4J**: 提供统一的日志接口
- **Logback**: 负责具体的日志输出处理
- **配置文件**: 控制日志的行为和输出

## 3. 日志级别详解

### 3.1 级别定义
日志级别从低到高依次为：TRACE < DEBUG < INFO < WARN < ERROR

#### TRACE (跟踪)
- **用途**: 最详细的日志信息，通常只在开发阶段使用
- **场景**: 方法进入/退出、变量值跟踪
- **示例**: `log.trace("进入方法 execute(), 参数: {}", param);`

#### DEBUG (调试)
- **用途**: 调试信息，用于开发阶段问题排查
- **场景**: SQL执行、业务流程跟踪
- **示例**: `log.debug("执行SQL: {}", sql);`

#### INFO (信息)
- **用途**: 应用程序正常运行的重要信息
- **场景**: 用户操作、系统状态变化
- **示例**: `log.info("用户登录成功, 用户名: {}", username);`

#### WARN (警告)
- **用途**: 警告信息，表示潜在问题但不影响正常运行
- **场景**: 配置缺失、性能问题
- **示例**: `log.warn("配置项未找到，使用默认值: {}", defaultValue);`

#### ERROR (错误)
- **用途**: 错误信息，表示系统出现了问题
- **场景**: 异常处理、系统故障
- **示例**: `log.error("数据库连接失败", exception);`

### 3.2 级别过滤
- **规则**: 只输出当前级别及以上的日志
- **示例**: 级别设置为INFO时，只输出INFO、WARN、ERROR

## 4. 使用方式介绍

### 4.1 基本使用

#### 添加注解
```java
import lombok.extern.slf4j.Slf4j;

@Slf4j
@Service
public class UserService {
    // 自动生成: private static final Logger log = LoggerFactory.getLogger(UserService.class);
}
```

#### 日志输出
```java
@Slf4j
public class UserService {
    public void createUser(String username) {
        // 不同级别的日志
        log.trace("创建用户开始, username: {}", username);
        log.debug("检查用户名是否已存在: {}", username);
        log.info("正在创建用户: {}", username);
        log.warn("用户名较短，建议修改: {}", username);
        log.error("创建用户失败", exception);
    }
}
```

### 4.2 参数化日志
```java
// 推荐：参数化日志，性能更好
log.info("用户 {} 在 {} 执行了 {} 操作", username, time, operation);

// 不推荐：字符串拼接，性能差
log.info("用户 " + username + " 在 " + time + " 执行了 " + operation + " 操作");
```

### 4.3 异常日志
```java
try {
    // 业务代码
} catch (Exception e) {
    // 记录异常信息，包含堆栈
    log.error("操作失败", e);
    
    // 或者只记录消息
    log.error("操作失败: {}", e.getMessage());
}
```

### 4.4 条件日志
```java
// 避免不必要的计算
if (log.isDebugEnabled()) {
    log.debug("复杂计算结果: {}", expensiveCalculation());
}

// 或者使用参数化日志（推荐）
log.debug("复杂计算结果: {}", expensiveCalculation());
```

- **当前文件**: `类型.log`

- **保留时间**: 30天
- **总大小限制**: 10GB
- **压缩**: 历史文件自动压缩

## 7. 配置文件详解

### 7.1 配置文件位置
- **文件名**: `logback-spring.xml`
- **位置**: `src/main/resources/`
- **优先级**: 高于`logback.xml`

### 7.2 主要配置项

#### 基础配置
```xml
<!-- 扫描配置文件变化 -->
<configuration scan="true" scanPeriod="60 seconds" debug="false">

<!-- 日志路径 -->
<property name="log.path" value="${LOG_PATH:-logs}"/>

<!-- 日志格式 -->
<property name="log.pattern" value="%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - [%method,%line] - %msg%n"/>

<!-- 文件管理 -->
<property name="log.maxHistory" value="30"/>
<property name="log.maxFileSize" value="50MB"/>
<property name="log.totalSizeCap" value="10GB"/>
```

#### 日志格式说明
- `%d{yyyy-MM-dd HH:mm:ss.SSS}`: 时间戳
- `[%thread]`: 线程名
- `%-5level`: 日志级别（左对齐，5字符宽度）
- `%logger{50}`: 类名（最多50字符）
- `[%method,%line]`: 方法名和行号
- `%msg`: 日志消息
- `%n`: 换行符

#### Appender配置
```xml
<!-- 控制台输出 -->
<appender name="console" class="ch.qos.logback.core.ConsoleAppender">
    <encoder>
        <pattern>${log.pattern}</pattern>
        <charset>UTF-8</charset>
    </encoder>
</appender>

<!-- 文件输出 -->
<appender name="file_info" class="ch.qos.logback.core.rolling.RollingFileAppender">
    <file>${log.path}/sys-info.log</file>
    <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
        <fileNamePattern>${log.path}/sys-info.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
        <maxFileSize>${log.maxFileSize}</maxFileSize>
        <maxHistory>${log.maxHistory}</maxHistory>
        <totalSizeCap>${log.totalSizeCap}</totalSizeCap>
    </rollingPolicy>
    <encoder>
        <pattern>${log.pattern}</pattern>
        <charset>UTF-8</charset>
    </encoder>
</appender>
```

#### 异步配置
```xml
<!-- 异步输出 -->
<appender name="async_info" class="ch.qos.logback.classic.AsyncAppender">
    <discardingThreshold>0</discardingThreshold>
    <queueSize>1024</queueSize>
    <appender-ref ref="file_info"/>
</appender>
```

#### Logger配置
```xml
<!-- SQL日志 -->
<logger name="com.zl.**.mapper" level="DEBUG" additivity="false">
    <appender-ref ref="console"/>
    <appender-ref ref="async_sql"/>
</logger>

<!-- 业务日志 -->
<logger name="com.zl.service" level="DEBUG" additivity="false">
    <appender-ref ref="console"/>
    <appender-ref ref="async_business"/>
</logger>
```

### 7.3 环境配置
```xml
<!-- 开发环境 -->
<springProfile name="dev">
    <logger name="com.zl" level="DEBUG"/>
    <root level="DEBUG">
        <appender-ref ref="console"/>
        <appender-ref ref="async_debug"/>
        <appender-ref ref="async_info"/>
        <appender-ref ref="async_warn"/>
        <appender-ref ref="async_error"/>
    </root>

<!-- 生产环境 -->
<springProfile name="prod">

    <root level="INFO">
        <appender-ref ref="async_info"/>

    </root>

```

## 9. 最佳实践

### 9.1 日志使用原则

#### DO - 推荐做法
```java
// 1. 使用参数化日志
log.info("用户 {} 执行了 {} 操作", username, operation);

// 2. 记录异常信息
log.error("操作失败", exception);

// 3. 使用合适的日志级别
log.debug("调试信息");
log.info("重要信息");
log.warn("警告信息");
log.error("错误信息");

// 4. 记录关键业务操作
log.info("订单创建成功, 订单号: {}, 金额: {}", orderId, amount);
```

#### DON'T - 避免做法
```java
// 1. 避免字符串拼接
log.info("用户 " + username + " 执行了 " + operation + " 操作");

// 2. 避免在生产环境输出DEBUG日志
log.debug("用户密码: " + password);

// 3. 避免记录敏感信息
log.info("用户信用卡号: " + creditCardNumber);

// 4. 避免在循环中输出大量日志
for (int i = 0; i < 10000; i++) {
    log.info("处理第 " + i + " 条记录");
}
```

### 9.2 性能优化

#### 异步日志
- 所有文件输出都配置为异步
- 队列大小设置为1024
- 丢弃阈值设置为0

#### 条件日志
```java
// 避免不必要的计算
if (log.isDebugEnabled()) {
    log.debug("复杂计算结果: {}", expensiveCalculation());
}
```

#### 日志级别控制
- 开发环境使用DEBUG级别
- 生产环境使用INFO级别
- 敏感操作使用WARN或ERROR级别

### 9.3 安全考虑

#### 敏感信息保护
```java
// 错误示例
log.info("用户登录: username={}, password={}", username, password);

// 正确示例
log.info("用户登录: username={}", username);
```

## 10. 常见问题

### 10.1 日志不输出
**问题**: 配置了日志但没有输出
**解决**:
1. 检查日志级别设置
2. 确认additivity="false"
3. 验证appender配置
4. 检查环境配置

### 10.2 日志文件过大
**问题**: 日志文件增长过快
**解决**:
1. 调整日志级别
2. 优化日志输出频率
3. 调整文件滚动策略
4. 增加总大小限制

### 10.3 性能问题
**问题**: 日志影响系统性能
**解决**:
1. 使用异步日志
2. 避免字符串拼接
3. 合理设置日志级别
4. 减少不必要的日志输出

### 10.4 日志丢失
**问题**: 异步日志导致日志丢失
**解决**:
1. 增加队列大小
2. 设置丢弃阈值
3. 监控队列状态
4. 考虑同步输出关键日志

### 10.5 环境配置不生效
**问题**: 不同环境配置没有生效
**解决**:
1. 检查spring.profiles.active
2. 确认配置文件命名
3. 验证springProfile配置
4. 检查配置文件加载顺序
