# 全局异常处理设计文档

## 1. 概述

本文档描述了智领软件公司基础脚手架项目的全局异常处理设计，包括异常处理架构、错误码管理、异常类继承体系以及使用方式。

## 2. 设计目标

- **统一性**：提供统一的异常处理机制和响应格式
- **可扩展性**：支持自定义异常类型和错误码
- **可维护性**：清晰的异常分类和错误码管理
- **兼容性**：保持对现有异常类的兼容性
- **国际化**：支持错误消息的国际化

## 3. 架构设计

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Controller Layer                      │
├─────────────────────────────────────────────────────────────┤
│                GlobalExceptionHandler                    │
│                     │                                │
│         ┌─────────────┴─────────────┐                  │
│         │                           │                  │
│    自定义异常处理               系统异常处理              │
│         │                           │                  │
├─────────────────┬───────────────────┴───────────────────┤
│                 │                                   │
│    BaseServiceException                    RuntimeException │
│         │                                   │
│   ┌─────┴─────┐                           │
│   │           │                           │
│业务异常    系统异常                        │
│   │           │                           │
│   │     ┌─────┴─────┐                     │
│   │     │           │                     │
│用户异常  文件异常    认证异常                │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 核心组件

#### 3.2.1 ErrorCode 错误码枚举

统一管理所有错误码，按功能模块分类：

- **通用错误码**：10000-19999
- **认证授权错误码**：20000-29999
- **用户相关错误码**：30000-39999
- **文件相关错误码**：40000-49999
- **业务错误码**：50000-59999
- **幂等性错误码**：60000-69999
- **限流错误码**：70000-79999

#### 3.2.2 BaseServiceException 统一异常基类

所有自定义异常的基类，提供以下特性：
- 统一的错误码和错误消息管理
- 支持错误参数和国际化
- 支持详细错误信息和模块标识
- 链式调用支持

#### 3.2.3 GlobalExceptionHandler 全局异常处理器

使用 `@RestControllerAdvice` 注解，统一处理所有异常：
- 自定义异常处理
- 参数校验异常处理
- 系统异常处理
- 兼容性异常处理

## 4. 异常类继承体系

### 4.1 继承关系图

```
RuntimeException
├── BaseServiceException (新增统一基类)
│   ├── AuthException (认证异常)
│   ├── DemoModeException (演示模式异常)
│   ├── UtilException (工具类异常)
│   ├── UserException (用户异常)
│   │   ├── UserPasswordNotMatchException
│   │   ├── UserPasswordRetryLimitExceedException
│   │   ├── CaptchaException
│   │   └── CaptchaExpireException
│   └── FileException (文件异常)
│       ├── FileSizeLimitExceededException
│       └── FileNameLengthLimitExceededException
├── ServiceException (兼容性保留)
├── GlobalException (兼容性保留)
└── BaseException (兼容性保留)
```

### 4.2 异常分类

#### 4.2.1 认证授权异常 (AuthException)
- 用于处理认证和授权相关的异常
- 错误码范围：20000-29999

#### 4.2.2 用户相关异常 (UserException)
- 用于处理用户操作相关的异常
- 错误码范围：30000-39999
- 包含密码错误、验证码错误等子类

#### 4.2.3 文件相关异常 (FileException)
- 用于处理文件操作相关的异常
- 错误码范围：40000-49999
- 包含文件大小、文件名长度等子类

#### 4.2.4 业务异常 (ServiceException)
- 用于处理一般业务逻辑异常
- 错误码范围：50000-59999

#### 4.2.5 工具类异常 (UtilException)
- 用于处理工具类操作异常
- 错误码范围：50000-59999

## 5. 错误码设计

### 5.1 错误码规则

错误码采用5位数字编码，规则如下：
- 前两位：模块标识
- 后三位：具体错误类型

### 5.2 错误码分类

| 模块 | 错误码范围 | 说明 |
|------|------------|------|
| 通用 | 10000-19999 | 系统通用错误 |
| 认证授权 | 20000-29999 | 认证和授权相关错误 |
| 用户管理 | 30000-39999 | 用户操作相关错误 |
| 文件管理 | 40000-49999 | 文件操作相关错误 |
| 业务逻辑 | 50000-59999 | 业务逻辑相关错误 |
| 幂等性 | 60000-69999 | 幂等性校验相关错误 |
| 限流 | 70000-79999 | 限流相关错误 |

### 5.3 常用错误码

| 错误码 | 错误消息 | 说明 |
|--------|----------|------|
| 200 | 操作成功 | 成功状态码 |
| 10000 | 系统异常 | 未知系统错误 |
| 10001 | 参数错误 | 请求参数错误 |
| 20001 | 未认证 | 用户未认证 |
| 20002 | 无权限 | 用户无权限访问 |
| 30002 | 密码不正确 | 用户密码错误 |
| 40003 | 文件大小超过限制 | 上传文件过大 |
| 50002 | 演示模式，不允许操作 | 演示环境限制 |

## 6. 异常处理流程

### 6.1 异常捕获流程

```
Controller抛出异常
        ↓
GlobalExceptionHandler捕获
        ↓
异常类型判断
        ↓
┌─────────────────┬─────────────────┬─────────────────┐
│   自定义异常    │   参数校验异常   │   系统异常      │
│ BaseServiceException │ BindException │ RuntimeException │
│                 │                 │                 │
│ 错误码提取       │ 字段错误提取     │ 默认错误码      │
│ 日志记录         │ 日志记录        │ 日志记录        │
│ 响应构建         │ 响应构建        │ 响应构建        │
└─────────────────┴─────────────────┴─────────────────┘
        ↓
统一响应格式返回
```

### 6.2 日志记录策略

- **自定义异常**：记录异常信息和堆栈
- **参数校验异常**：记录字段验证错误详情
- **系统异常**：记录完整异常堆栈信息
- **请求信息**：记录请求URI、方法和参数

## 10. 最佳实践

### 10.1 异常使用原则

1. **精确性**：使用最具体的异常类型
2. **有意义**：提供清晰的错误消息
3. **可操作**：错误信息应帮助用户解决问题
4. **一致性**：相同场景使用相同的错误码

### 10.2 异常抛出建议

```java
// 推荐：使用具体的异常类型和错误码
if (passwordNotMatch) {
    throw new UserPasswordNotMatchException();
}

// 推荐：使用带参数的异常
if (retryCount > maxRetry) {
    throw new UserPasswordRetryLimitExceedException(maxRetry, lockTime);
}

// 避免：直接抛出RuntimeException
throw new RuntimeException("操作失败");
```

### 10.3 异常处理建议

```java
// 推荐：在Controller中使用try-catch处理业务异常
try {
    // 业务逻辑
    return Result.success(data);
} catch (UserException e) {
    // 记录日志
    log.warn("用户操作失败: {}", e.getMessage());
    // 重新抛出，由全局异常处理器处理
    throw e;
}
```

## 11. 扩展指南

### 11.1 新增异常类型

1. 继承BaseServiceException
2. 定义对应的ErrorCode枚举值
3. 在GlobalExceptionHandler中添加处理方法（如需特殊处理）

### 11.2 新增错误码

1. 在ErrorCode枚举中添加新的错误码
2. 确保错误码在指定范围内
3. 添加对应的国际化消息

### 11.3 自定义异常处理

```java
@ExceptionHandler(CustomException.class)
public ResponseEntity<Result<Object>> handleCustomException(CustomException exception, HttpServletRequest request) {
    logException(exception, request);
    return ResponseEntity.ok(Result.fail(exception.getCode(), exception.getMessage()));
}
```
