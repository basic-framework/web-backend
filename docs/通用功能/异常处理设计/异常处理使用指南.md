# 异常处理使用指南

## 1. 快速开始

### 1.1 抛出业务异常

```java
// 使用预定义的错误码
throw new UserPasswordNotMatchException();

// 使用自定义错误消息
throw new UserPasswordNotMatchException("用户密码不匹配");

// 使用带参数的错误码
throw new UserPasswordRetryLimitExceedException(5, 30);
```

### 1.2 抛出通用业务异常

```java
// 使用错误码枚举
throw new BaseServiceException(ErrorCode.PARAM_ERROR);

// 使用错误码和消息
throw new BaseServiceException(ErrorCode.PARAM_ERROR, "用户名不能为空");

// 使用错误码和参数
throw new BaseServiceException(ErrorCode.USER_PASSWORD_RETRY_LIMIT_EXCEED, 
                              new Object[]{5, 30});
```

### 1.3 在Controller中使用

```java
@RestController
@RequestMapping("/api/user")
public class UserController {
    
    @PostMapping("/login")
    public Result<UserVO> login(@RequestBody LoginDTO loginDTO) {
        try {
            UserVO userVO = userService.login(loginDTO);
            return Result.success(userVO);
        } catch (UserException e) {
            // 记录日志
            log.warn("用户登录失败: {}", e.getMessage());
            // 重新抛出，由全局异常处理器处理
            throw e;
        }
    }
}
```

## 4. 异常处理最佳实践

### 4.1 异常使用原则

1. **使用具体异常类型**
   ```java
   // 推荐
   throw new UserPasswordNotMatchException();
   
   // 不推荐
   throw new BaseServiceException(ErrorCode.USER_PASSWORD_NOT_MATCH);
   ```

2. **提供有意义的错误消息**
   ```java
   // 推荐
   throw new UserPasswordNotMatchException("用户名或密码错误");
   
   // 不推荐
   throw new UserPasswordNotMatchException("错误");
   ```

3. **使用参数化错误消息**
   ```java
   // 推荐
   throw new UserPasswordRetryLimitExceededException(5, 30);
   
   // 不推荐
   throw new UserPasswordRetryLimitExceededException("密码错误次数超过5次，锁定30分钟");
   ```

### 4.2 异常处理策略

1. **Controller层异常处理**
   ```java
   @PostMapping("/create")
   public Result<OrderVO> createOrder(@RequestBody CreateOrderDTO dto) {
       // 业务逻辑
       OrderVO orderVO = orderService.createOrder(dto);
       return Result.success(orderVO);
       // 不需要try-catch，让全局异常处理器处理
   }
   ```

2. **Service层异常处理**
   ```java
   @Service
   public class OrderServiceImpl implements OrderService {
       
       public OrderVO createOrder(CreateOrderDTO dto) {
           // 参数校验
           if (dto.getUserId() == null) {
               throw new BaseServiceException(ErrorCode.PARAM_MISSING, "用户ID不能为空");
           }
           
           // 业务逻辑
           try {
               Order order = orderRepository.save(dto);
               return OrderVO.from(order);
           } catch (DataAccessException e) {
               log.error("保存订单失败", e);
               throw new BaseServiceException(ErrorCode.OPERATION_FAILED, "创建订单失败");
           }
       }
   }
   ```

3. **Repository层异常处理**
   ```java
   @Repository
   public class OrderRepositoryImpl implements OrderRepository {
       
       public Order findById(Long id) {
           try {
               return orderMapper.selectById(id);
           } catch (Exception e) {
               log.error("查询订单失败, id: {}", id, e);
               throw new BaseServiceException(ErrorCode.SYSTEM_ERROR, "查询订单失败");
           }
       }
   }
   ```

## 5. 异常处理进阶

### 5.1 异常链

```java
try {
    // 可能抛出异常的操作
    processOrder(order);
} catch (SQLException e) {
    // 包装底层异常
    throw new BaseServiceException(ErrorCode.DATABASE_ERROR, "处理订单失败")
            .setDetailMessage(e.getMessage())
            .setModule("order");

```

```java
// 创建异常分组
public class ValidationException extends BaseServiceException {
    public ValidationException(List<String> errors) {
        super(ErrorCode.PARAM_FORMAT_ERROR, String.join("; ", errors));
    }
}

// 使用
List<String> errors = validateOrder(order);
if (!errors.isEmpty()) {
    throw new ValidationException(errors);
}
```

### 5.3 异常恢复

```java
@Retryable(value = {ServiceException.class}, maxAttempts = 3, backoff = @Backoff(delay = 1000))

    // 可能失败的业务逻辑

}

@Recover

    log.error("创建订单失败，进入恢复模式", e);
    // 执行恢复逻辑
    return createOrderInRecoveryMode(dto);
}
```

## 7. 异常监控

### 7.1 异常日志记录

```java

// 业务层可以记录特定业务异常

    processPayment(payment);

    // 记录业务异常

             payment.getOrderId(), payment.getAmount(), e.getMessage());

```

### 7.2 异常指标收集

```java
// 使用Micrometer收集异常指标
@Component
public class ExceptionMetrics {
    
    private final Counter exceptionCounter;
    
    public ExceptionMetrics(MeterRegistry meterRegistry) {
        this.exceptionCounter = Counter.builder("business.exceptions")
                .description("Business exception count")
                .register(meterRegistry);
    }
    
    public void recordException(String exceptionType) {
        exceptionCounter.increment(Tags.of("type", exceptionType));
    }
}

// 在异常处理器中使用
@ExceptionHandler(BusinessException.class)
public ResponseEntity<Result<Object>> handleBusinessException(BusinessException exception, HttpServletRequest request) {
    // 记录异常指标
    exceptionMetrics.recordException(exception.getClass().getSimpleName());
    
    // 处理异常
    logException(exception, request);
    return ResponseEntity.ok(Result.fail(exception.getCode(), exception.getMessage()));
}
```

## 8. 常见问题

### 8.1 Q: 如何选择合适的异常类型？

A: 根据业务场景选择最具体的异常类型：
- 用户相关操作：使用UserException及其子类
- 文件操作：使用FileException及其子类
- 认证授权：使用AuthException
- 一般业务逻辑：使用BaseServiceException
- 工具类操作：使用UtilException

### 8.2 Q: 如何处理国际化错误消息？

A: 使用ErrorCode枚举和MessageUtils：
```java
// 在异常中使用
throw new UserPasswordNotMatchException();

// 在国际化配置文件中
# messages.properties
user.password.not.match=密码不正确

# messages_en.properties
user.password.not.match=Password not match
```

### 8.3 Q: 如何在异常中传递额外信息？

A: 使用detailMessage字段：
```java
BaseServiceException exception = new BaseServiceException(ErrorCode.OPERATION_FAILED)
    .setDetailMessage("数据库连接超时")
    .setModule("order");
```

### 8.4 Q: 如何处理第三方库异常？

A: 包装为业务异常：
```java
try {
    // 第三方库调用
    externalService.process(data);
} catch (ExternalServiceException e) {
    throw new BaseServiceException(ErrorCode.BUSINESS_ERROR, 
                                  "第三方服务调用失败")
            .setDetailMessage(e.getMessage());
}
```

## 9. 迁移指南

### 9.1 从旧异常体系迁移

1. **替换异常基类**
   ```java
   // 旧代码
   throw new ServiceException("操作失败");
   
   // 新代码
   throw new BaseServiceException(ErrorCode.OPERATION_FAILED, "操作失败");
   ```

2. **使用错误码枚举**
   ```java
   // 旧代码
   throw new UserException("user.not.found");
   
   // 新代码
   throw new UserException(ErrorCode.USER_NOT_FOUND);
   ```

3. **更新异常处理**
   ```java
   // 旧代码
   @ExceptionHandler(UserException.class)
   public ResponseEntity<?> handleUserException(UserException e) {
       return ResponseEntity.badRequest().body(e.getMessage());
   }
   
   // 新代码（由GlobalExceptionHandler统一处理）
   // 不需要单独处理，除非需要特殊逻辑
   ```

### 9.2 渐进式迁移策略

1. **第一阶段**：新功能使用新异常体系
2. **第二阶段**：关键模块迁移到新异常体系
3. **第三阶段**：全面迁移并移除旧异常类
