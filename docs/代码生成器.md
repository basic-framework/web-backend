# ruoyi-generator 代码生成器模块架构设计文档

## 1. 模块概述

### 1.1 模块定位
`ruoyi-generator` 是 RuoYi-AI 项目中的代码生成模块，基于 **Schema 驱动** 的设计理念，通过可视化配置数据模型，自动生成符合项目规范的后端代码和前端代码。

### 1.2 核心特性
- **Schema 驱动**：通过数据模型定义来驱动代码生成，而非直接从数据库表读取
- **模板化生成**：基于 Apache Velocity 模板引擎，支持高度定制化
- **多数据库支持**：支持 MySQL、Oracle、PostgreSQL、SQL Server
- **前后端分离**：同时支持后端 Java 代码和前端代码生成
- **事件驱动**：通过 Spring Event 机制实现模型变更的自动处理

**文件路径**: `org.ruoyi.generator.config.GenConfig`

```velocity

import com.baomidou.mybatisplus.annotation.*;
import lombok.Data;
#foreach ($import in $importList)
import ${import};
#end

/**
 * ${functionName}对象 ${tableName}
 *
 * @author ${author}
 * @date ${datetime}
 */
@Data
@TableName("${tableName}")
public class ${ClassName} implements Serializable {
#foreach ($column in $columns)
    /**
     * $column.columnComment
     */
    #if($column.isPk==1)
    @TableId(value = "${column.columnName}", type = IdType.AUTO)
    #end
    private $column.javaType $column.javaField;
#end
}
```

## 8. 使用场景

### 8.1 后端代码生成

1. 在数据库中创建表
2. 通过 Schema 管理界面创建数据模型，关联表名
3. 系统自动读取表结构，生成 SchemaField 数据
4. 调用 `/tool/gen/batchGenCode?tableNameStr=表名` 接口生成代码
5. 生成的代码输出到项目 classpath

**生成的文件结构**:
```
src/main/java/
└── org/ruoyi/aihuman/
    ├── domain/
    │   ├── {ClassName}.java           # 实体类
    │   ├── vo/{ClassName}Vo.java      # 视图对象
    │   └── bo/{ClassName}Bo.java      # 业务对象
    ├── mapper/
    │   └── {ClassName}Mapper.java     # Mapper 接口
    ├── service/
    │   ├── I{ClassName}Service.java   # Service 接口
    │   └── impl/{ClassName}ServiceImpl.java
    └── controller/
        └── {ClassName}Controller.java # Controller

src/main/resources/
└── mapper/
    └── {ModuleName}/
        └── {ClassName}Mapper.xml      # MyBatis 映射文件
```

### 8.2 前端代码生成

1. 后端代码生成完成后
2. 调用 `/tool/gen/batchGenFrontendCode` 接口
3. 传入前端项目路径和构建命令
4. 系统执行命令生成前端代码

**接口参数**:
- `workPath`: 前端项目根目录
- `previewCode`: 执行的生成命令（如：`npm run gen:vue`）

### 8.3 自定义扩展

1. 修改 `generator.yml` 配置文件
2. 在 `vm/` 目录下添加自定义模板
3. 修改 `VelocityUtils.getTemplateList()` 添加新模板

## 11. 注意事项

1. **开发环境限制**: `GenController` 使用了 `@Profile("dev")` 注解，仅在开发环境可用
2. **表名前缀处理**: 配置 `autoRemovePre: true` 可自动去除表前缀
3. **主键处理**: 若未指定主键字段，默认使用第一个字段作为主键
4. **文件覆盖**: 生成的文件会直接覆盖同名文件，请谨慎操作

该模块是 RuoYi-AI 项目提升开发效率、保证代码质量的重要工具。
