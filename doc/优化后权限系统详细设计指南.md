## 问题背景与分析

### 原始问题
**核心问题**：在Filter中清除ThreadLocal中信息，在AuthorizationManager中存ThreadLocal信息，存在执行顺序冲突。

### 问题分析
```
执行顺序问题：
Filter (清除ThreadLocal) → AuthorizationManager (设置ThreadLocal)
         ↑                                    ↓
      清除操作                          设置操作
         ↓                                    ↑
   ThreadLocal为空 ← 权限验证无法获取用户信息
```

### 根本原因
1. **执行时序错误**：Filter在AuthorizationManager之前执行，导致ThreadLocal被过早清除
2. **职责混乱**：多个组件都操作ThreadLocal，生命周期管理不统一
3. **重复验证**：Filter和AuthorizationManager都进行JWT解析，造成性能浪费

### 认证流程代码实现

#### JwtAuthenticationFilter核心逻辑
```java
@Override
protected void doFilterInternal(HttpServletRequest request,
                                HttpServletResponse response,
                                FilterChain filterChain) {
    try {
        // 步骤1: 提取并验证令牌
        String jwtToken = extractAndValidateToken(request);
        if (jwtToken == null) {
            filterChain.doFilter(request, response);
            return;
        }

        // 步骤2: 解析JWT获取用户信息
        Claims claims = jwtUtil.parseToken(jwtToken);

        // 步骤3: 构建用户认证信息
        Authentication authentication = createAuthentication(claims, request);

        // 步骤4: 设置认证信息到安全上下文
        SecurityContextHolder.getContext().setAuthentication(authentication);

        log.debug("JWT认证成功，用户: {}", claims.getSubject());
    } catch (Exception e) {
        log.error("JWT认证处理异常", e);
        SecurityContextHolder.clearContext();
    }

    filterChain.doFilter(request, response);
}
```

#### UserTokenInterceptor核心逻辑
```java
@Override
public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
    // 非控制器请求直接放行
    if(!(handler instanceof HandlerMethod)){
        return true;
    }
    
    // 步骤1: 从头部获取userToken
    String userToken = request.getHeader("Authorization");
    if(!ObjectUtil.isEmpty(userToken)){
        // 步骤2: 构建Redis key
        String jwtTokenKey = UserConstant.JWT_TOKEN + userToken;
        
        // 步骤3: 从Redis获取JWT令牌
        String jwtToken = stringRedisTemplate.opsForValue().get(jwtTokenKey);
        
        if(!ObjectUtil.isEmpty(jwtToken)){
            // 步骤4: 解析JWT令牌
            Map<String, Object> claims = jwtUtil.parseToken(jwtToken);
            Object userObj = claims.get("currentUser");
            String currentUser = String.valueOf(userObj);
            
            // 步骤5: 将用户信息存入ThreadLocal
            UserThreadLocal.setSubject(currentUser);
        }
    }
    return true;
}

@Override
public void afterCompletion(HttpServletRequest request, HttpServletResponse response, 
                       Object handler, Exception ex) {
    // 步骤6: 移除当前线程变量中的数据
    UserThreadLocal.remove();
}
```

#### JwtAuthorizationManager核心逻辑
```java
@Override
public AuthorizationDecision check(Supplier<Authentication> authentication,
                                   RequestAuthorizationContext requestContext) {
    try {
        // 步骤1: 提取请求信息
        String requestMethodAndPath = extractRequestInfo(requestContext);

        // 步骤2: 从ThreadLocal获取用户信息
        LoginVo loginVo = getUserFromThreadLocal();
        if (loginVo == null) {
            log.warn("ThreadLocal中未找到用户信息，路径: {}", requestMethodAndPath);
            return new AuthorizationDecision(false);
        }

        // 步骤3: 验证资源访问权限
        if (hasResourceAccessPermission(loginVo, requestMethodAndPath)) {
            log.debug("用户{}有权访问资源: {}", loginVo.getUsername(), requestMethodAndPath);
            return new AuthorizationDecision(true);
        }

        log.warn("用户{}无权访问资源: {}", loginVo.getUsername(), requestMethodAndPath);
        return new AuthorizationDecision(false);

    } catch (Exception e) {
        log.error("授权校验过程发生异常", e);
        return new AuthorizationDecision(false);
    }
}

private boolean hasResourceAccessPermission(LoginVo loginVo, String requestMethodAndPath) {
    // 步骤4: 获取用户权限列表
    Set<String> resourcePaths = loginVo.getResourcePaths();
    if (resourcePaths == null || resourcePaths.isEmpty()) {
        return false;
    }

    // 步骤5: 使用AntPathMatcher匹配资源路径
    return resourcePaths.stream()
            .anyMatch(resourcePath -> antPathMatcher.match(resourcePath, requestMethodAndPath));
}
```

## 核心组件实现原理

### 1. JwtAuthenticationFilter实现原理

#### 设计原理
- **过滤器模式**：实现Spring Security的OncePerRequestFilter
- **职责单一**：只负责JWT解析和SecurityContext设置
- **无状态设计**：不依赖ThreadLocal，避免时序问题

#### 核心实现
```java
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) {
        // 1. 令牌提取和验证
        String jwtToken = extractAndValidateToken(request);
        
        // 2. JWT解析
        Claims claims = jwtUtil.parseToken(jwtToken);
        
        // 3. 构建Authentication对象
        Authentication authentication = createAuthentication(claims, request);
        
        // 4. 设置到SecurityContext
        SecurityContextHolder.getContext().setAuthentication(authentication);
        
        // 5. 继续过滤器链
        filterChain.doFilter(request, response);
    }
}
```

### 2. UserTokenInterceptor实现原理

#### 设计原理
- **拦截器模式**：实现Spring MVC的HandlerInterceptor
- **生命周期管理**：preHandle设置，afterCompletion清理
- **线程安全**：每个线程独立的ThreadLocal存储

#### 核心实现
```java
public class UserTokenInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        // 1. 请求前处理：设置ThreadLocal
        String userToken = extractUserToken(request);
        if (isValidToken(userToken)) {
            LoginVo userInfo = parseUserInfo(userToken);
            UserThreadLocal.setSubject(JSONObject.toJSONString(userInfo));
            UserThreadLocal.set(userInfo.getId());
        }
        return true;
    }
    
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, 
                           Object handler, Exception ex) {
        // 2. 请求后处理：清理ThreadLocal
        UserThreadLocal.remove();
    }
}
```

### 3. JwtAuthorizationManager实现原理

#### 设计原理
- **授权管理器**：实现Spring Security的AuthorizationManager
- **权限验证**：专注于资源访问权限检查
- **高效读取**：从ThreadLocal获取用户信息，避免重复解析

#### 核心实现
```java
public class JwtAuthorizationManager implements AuthorizationManager<RequestAuthorizationContext> {
    
    @Override
    public AuthorizationDecision check(Supplier<Authentication> authentication,
                                   RequestAuthorizationContext requestContext) {
        // 1. 从ThreadLocal获取用户信息
        LoginVo loginVo = getUserFromThreadLocal();
        
        // 2. 权限验证
        String requestPath = extractRequestPath(requestContext);
        boolean hasPermission = checkResourcePermission(loginVo, requestPath);
        
        // 3. 返回授权决策
        return new AuthorizationDecision(hasPermission);
    }
}
```

#### 权限加载流程

#### 权限加载实现
```java
@Service
public class LoginServiceImpl implements LoginService {
    
    @Override
    public LoginVo login(LoginDto loginDto) {
        // 1. 验证用户名密码
        User user = authenticateUser(loginDto.getUsername(), loginDto.getPassword());
        
        // 2. 查询用户角色
        List<Role> roles = roleService.getRolesByUserId(user.getId());
        
        // 3. 查询角色资源
        Set<Resource> resources = new HashSet<>();
        for (Role role : roles) {
            List<Resource> roleResources = resourceService.getResourcesByRoleId(role.getId());
            resources.addAll(roleResources);
        }
        
        // 4. 构建权限对象
        LoginVo loginVo = buildLoginVo(user, roles, resources);
        
        // 5. 生成JWT令牌
        String jwtToken = generateJwtToken(loginVo);
        
        // 6. 存储到Redis
        storeTokenToRedis(jwtToken, loginVo);
        
        return loginVo;
    }
    
    private LoginVo buildLoginVo(User user, List<Role> roles, Set<Resource> resources) {
        LoginVo loginVo = new LoginVo();
        loginVo.setId(user.getId());
        loginVo.setUsername(user.getUsername());
        loginVo.setNickname(user.getNickname());
        
        // 设置角色信息
        Set<String> roleCodes = roles.stream()
                .map(Role::getRoleCode)
                .collect(Collectors.toSet());
        loginVo.setRoles(roleCodes);
        
        // 设置资源路径
        Set<String> resourcePaths = resources.stream()
                .map(Resource::getResourcePath)
                .collect(Collectors.toSet());
        loginVo.setResourcePaths(resourcePaths);
        
        return loginVo;
    }
}
```

### 3. 权限验证机制

#### 路径匹配原理
```java
public class PathPermissionMatcher {
    private final AntPathMatcher pathMatcher = new AntPathMatcher();
    
    /**
     * 验证用户是否有访问指定路径的权限
     */
    public boolean hasPermission(LoginVo loginVo, String requestPath) {
        Set<String> resourcePaths = loginVo.getResourcePaths();
        
        return resourcePaths.stream()
                .anyMatch(resourcePath -> {
                    // 支持通配符匹配
                    // 例如：/api/user/** 可以匹配 /api/user/list, /api/user/info等
                    return pathMatcher.match(resourcePath, requestPath);
                });
    }
}
```

#### 权限验证示例
```java

public class OptimizedUserThreadLocal {
    // 使用弱引用避免内存泄漏
    private static final ThreadLocal<WeakReference<LoginVo>> userThreadLocal = 
        ThreadLocal.withInitial(() -> new WeakReference<>(null));
    
    public static void setUser(LoginVo user) {
        userThreadLocal.set(new WeakReference<>(user));
    }
    
    public static LoginVo getUser() {
        WeakReference<LoginVo> ref = userThreadLocal.get();
        return ref != null ? ref.get() : null;
    }
    
    public static void remove() {
        userThreadLocal.remove();
    }
```
