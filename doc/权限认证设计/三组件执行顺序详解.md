# 三组件执行顺序详解

## 问题分析

你提出的问题非常关键：**"在Filter中清除ThreadLocal中信息，在AuthorizationManager中存ThreadLocal信息，不是要先过Filter吗？"**

这个问题确实揭示了Spring Security过滤器链和Spring MVC拦截器的执行顺序问题。

## 问题所在

### 1. 原始问题分析
```
原始问题代码执行顺序：
1. JwtAuthenticationFilter (清除ThreadLocal) ← 问题：ThreadLocal还未设置
2. JwtAuthorizationManager (设置ThreadLocal) ← 问题：设置后可能立即被清除
3. Controller业务逻辑 ← 问题：无法获取ThreadLocal中的用户信息
```

### 2. 当前解决方案验证
```
当前优化后执行顺序：
1. JwtAuthenticationFilter (不操作ThreadLocal) ← 正确：只负责JWT解析和SecurityContext
2. UserTokenInterceptor.preHandle (设置ThreadLocal) ← 正确：在Controller前设置
3. Controller业务逻辑 ← 正确：可正常获取ThreadLocal
4. JwtAuthorizationManager (从ThreadLocal读取) ← 正确：权限验证时读取
5. UserTokenInterceptor.afterCompletion (清除ThreadLocal) ← 正确：请求完成后清理
```

## 潜在问题与解决方案

### 1. JwtAuthorizationManager执行时机问题

#### 问题分析
```java
// SecurityConfig中的配置
.access(jwtAuthorizationManager)
```

这个配置意味着`JwtAuthorizationManager.check()`会在Spring Security过滤器链中执行，**而不是在Controller方法执行后**。

#### 实际执行顺序
```
1. JwtAuthenticationFilter.doFilterInternal()
2. JwtAuthorizationManager.check() ← 在这里执行，此时ThreadLocal还未设置
3. UserTokenInterceptor.preHandle()
4. Controller方法执行
```

#### 解决方案
需要调整配置，确保JwtAuthorizationManager在Controller执行后执行：

```java
// 方案1: 移除SecurityConfig中的access配置，使用注解方式
@Configuration
@EnableGlobalMethodSecurity(prePostEnabled = true)
public class SecurityConfig {
    // 移除.access(jwtAuthorizationManager)
    // 使用@PreAuthorize注解进行权限控制
}

// 方案2: 自定义拦截器进行权限验证
@Component
public class PermissionInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        // 在这里进行权限验证
        return true;
    }
}
```

### 2. 推荐的完整解决方案

#### 修改SecurityConfig
```java
@Configuration
@EnableGlobalMethodSecurity(prePostEnabled = true)
public class SecurityConfig {
    
    @Autowired
    private JwtAuthenticationFilter jwtAuthenticationFilter;
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        List<String> ignoreUrl = securityConfigProperties.getIgnoreUrl();
        
        http
            .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class)
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers(ignoreUrl.toArray(new String[0]))
                .permitAll()
                .anyRequest()
                .authenticated()  // 只需要认证，不需要在这里进行授权
            )
            .csrf(csrf -> csrf.disable())
            .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS));
        
        return http.build();
    }
}
```

#### 修改Controller使用注解
```java
@RestController
```

#### 保留UserTokenInterceptor
```java
@Component
public class UserTokenInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        // 设置ThreadLocal用户信息
        String userToken = request.getHeader("Authorization");
        if (!ObjectUtil.isEmpty(userToken)) {
            String jwtTokenKey = UserConstant.JWT_TOKEN + userToken;
            String jwtToken = stringRedisTemplate.opsForValue().get(jwtTokenKey);
            
            if (!ObjectUtil.isEmpty(jwtToken)) {
                Map<String, Object> claims = jwtUtil.parseToken(jwtToken);
                Object userObj = claims.get("currentUser");
                String currentUser = String.valueOf(userObj);
                
                UserThreadLocal.setSubject(currentUser);
            }
        }
        return true;
    }
    
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, 
                           Object handler, Exception ex) {
        UserThreadLocal.remove();
    }
}
```

## 总结

### 关键点
1. **JwtAuthorizationManager在SecurityConfig中的配置会导致它在过滤器链中执行**，此时ThreadLocal还未设置
2. **UserTokenInterceptor在Controller前执行**，可以正确设置ThreadLocal
3. **权限验证应该通过@PreAuthorize注解在Controller方法执行时进行**，而不是在过滤器链中

### 推荐方案
1. **移除SecurityConfig中的.access(jwtAuthorizationManager)配置**
2. **使用@EnableGlobalMethodSecurity启用方法级权限控制**
3. **在Controller方法上使用@PreAuthorize注解进行权限验证**
4. **保留UserTokenInterceptor进行ThreadLocal生命周期管理**

这样可以确保ThreadLocal在权限验证前正确设置，并在请求完成后正确清理。
